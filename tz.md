Тестовое задание: Разработка веб-приложения для отображения погоды
Описание:
Необходимо разработать веб-приложение, включающее систему аутентификации пользователей и возможность отображения погодной информации с использованием графиков. Приложение должно реализовать следующие требования:
1. Форма логина/пароля:
•	Реализовать страницу с формой для входа (логин и пароль).
•	Использовать локальное хранилище (localStorage) для хранения токена авторизации после успешного логина.
2. Завести пользователя с доступом на чтение страницы с погодой:
•	Создать функционал для добавления нового пользователя.
•	После успешной регистрации пользователю должно быть предоставлено право на доступ к странице с погодой.
3. Незалогиненный пользователь не может читать страницы (403 error):
•	Реализовать защиту всех страниц (кроме страницы логина/регистрации) от неавторизованных пользователей.
•	Если пользователь не залогинен, он должен получать ошибку 403 при попытке доступа к защищённым страницам.
4. Графики погоды:
•	Реализовать на странице компонент для отображения графиков погоды.
o	Для визуализации можно использовать Plotly.js, Chart.js, Recharts, D3.js, ECharts
5. Анализ данных о погоде с помощью нескольких графиков:
•	Реализовать несколько типов графиков для анализа данных о погоде:
o	Линейный график (для отображения изменений температуры с течением времени).
o	Гистограмма (для анализа распределения температуры за период).
o	График со скользящей средней (для анализа трендов температуры).
6. Совмещение двух графиков (Температура и Влажность):
•	Реализовать график, который одновременно отображает:
o	Температуру (ось Y1).
o	Влажность (ось Y2).
•	Обе переменные должны отображаться на одном графике, но с разными осями Y, чтобы было удобно сравнивать изменения температуры и влажности во времени.
7. Интерактивные графики:
•	Реализовать возможность взаимодействия с графиками:
o	Выбор диапазона времени для отображения данных.
o	Настройка типов графиков (пользователь может переключаться между разными видами графиков).
o	Отображение значений при наведении на точки графика.
8. Тема оформления графиков:
•	Реализовать переключение между светлой и тёмной темой для графиков.

Требования:
•	Frontend: React, Typescript, Eslint, Prettier.
•	Графики: Plotly.js, Chart.js, Recharts, D3.js, ECharts (на выбор).
•	Авторизация: локальное хранилище (localStorage).
•	Компоненты можно использовать любые (предпочтительно open-source).
•	Код должен быть хорошо структурированным и сопровождаться комментариями, где это необходимо.
Дополнительно:
•	Если в приложении будет интеграция с реальным API для получения данных о погоде (например, OpenWeather), это будет большим плюсом.
•	Сохранение пользовательских настроек между перезагрузками страницы (например, тему, выбранный тип графика, выбранный диапазон дат или последнюю выбранную локацию)
•	Использование единого подхода к управлению состоянием приложения


Оценка задания:
•	Корректность работы формы логина и защиты страниц.
•	Правильность и разнообразие графиков.
•	Интерактивность и кастомизация графиков.
•	Корректное отображение данных для двух осей (температура и влажность).
•	Чистота,  структурированность кода и разделение ответственности в коде.
